# spec.md

## Goal

Build **Loamwake**, a fully playable fixed-room ecological puzzle platformer prototype in Python 3.11 with Pygame, where death rotates the player through three soil creatures and reshapes the terrain.

## Problem

The original idea had a strong mechanic but risked becoming a generic side-scroller. The stronger version is a room-based puzzle platformer where movement, creature identity, death, and terrain transformation are the whole game.

## Users

- Primary user: a solo player who likes strange puzzle platformers and ecological game themes.
- Builder user: a non-developer who wants Codex to create a complete local prototype.

## MVP Scope

Version 1 includes:

- Python 3.11.
- Pygame.
- 1280x720 window.
- 60 FPS.
- Fixed-room / single-screen puzzle platformer format.
- Tile-based rooms defined as text grids in `levels.py`.
- Three playable hosts: Springtail, Dung Beetle, Earthworm.
- Host switching only through death.
- Fixed rotation: Springtail → Dung Beetle → Earthworm → Springtail.
- Tile collision.
- Host-specific movement.
- Host-specific terrain interaction.
- Hazards.
- Death-morph terrain.
- Three-death room reset.
- Room progression.
- Win screen.
- Restart option.
- HUD showing host name and deaths in current room.
- Simple generated or procedural sounds.
- Asset loading from `/assets`.
- Rectangle fallback rendering if generated images are missing.
- Clear code modules with comments for a non-developer.

## Non-Goals

Version 1 does not include online play, save files, enemies, procedural generation, inventory, upgrades, complex animation trees, scrolling camera levels, executable packaging, GitHub setup, or external paid assets.

## Recommended Stack

- Runtime: Python 3.11.
- Game library: Pygame.
- Assets: PNG images in `/assets`, generated by Codex when possible.
- Levels: text grids in `levels.py`.
- Sounds: generated simple wave tones or Pygame mixer sounds stored in `/assets/sounds`.
- Testing: lightweight Python tests for level parsing and terrain mutation logic where practical.
- Deployment: local run only.

## Game Format

Loamwake is a **fixed-room puzzle platformer**, not a side-scrolling platformer.

Each room fits the 1280x720 window. Do not implement a scrolling camera for MVP.

The room should feel like an illustrated soil diorama: roots, pore tunnels, cracked clods, fungal hazards, and compacted earth.

The tile grid controls collision and gameplay. Images are visual only.

## Tile Characters

| Character | Meaning | Behavior |
|---|---|---|
| `X` | solid soil | solid for all hosts |
| `C` | cracked soil | solid until broken by Beetle landing |
| `G` | pore gap | passable only by Springtail; solid/blocking for Beetle and Earthworm |
| `S` | soft soil | diggable only by Earthworm while holding down plus direction |
| `H` | hazard | damages hosts after immunity rules |
| `P` | player spawn | spawn location, not solid |
| `E` | exit | loads next room |
| `.` | empty | passable |

## Hosts

### Springtail

- Visual: cyan creature/sprite or cyan rectangle fallback.
- Speed: 8 px/frame.
- Cannot jump.
- Has horizontal dash.
- Dash distance: 300 px burst.
- Dash cooldown: 1 second.
- Can pass through `G` pore gap tiles.
- Cannot pass through `X`, `C`, or `S`.
- Death morph effect: converts 3x3 tile area centered on death point into empty tunnel tiles (`.`).

### Dung Beetle

- Visual: brown creature/sprite or brown rectangle fallback.
- Speed: 3 px/frame.
- Has high jump clearing roughly 3 tiles of height.
- Cannot pass through `G`.
- Breaks `C` cracked tiles by landing on them.
- Broken `C` tiles become empty (`.`) permanently within the room until reset.
- Death morph effect: converts 3x3 tile area centered on death point into solid platform tiles (`X`).

### Earthworm

- Visual: pink creature/sprite or pink rectangle fallback.
- Speed: 4 px/frame.
- No jump.
- No dash.
- Tunnels through `S` soft soil by holding down plus a direction.
- Tunneled `S` tiles become empty (`.`) permanently within the room until reset.
- Immune to hazard damage for the first 2 seconds of continuous hazard contact.
- After 2 seconds of hazard contact, takes normal hazard damage/death.
- Death morph effect: converts 3x3 tile area centered on death point into soft soil tiles (`S`).

## Death-Morph Mechanic

This is the core feature. Do not simplify it.

When the player touches a hazard past any host immunity window:

1. Convert the 3x3 tile area centered on the death point into new terrain:
   - Springtail corpse = empty tunnel tiles (`.`)
   - Beetle corpse = solid platform tiles (`X`)
   - Earthworm corpse = soft soil tiles (`S`)
2. Respawn at `P` as the next host in the fixed rotation.
3. Corpse terrain persists across deaths within the same room.
4. Broken cracked tiles and dug soft soil also persist within the same room.
5. Three deaths in one room triggers a full room reset:
   - reset corpse terrain
   - reset broken cracked tiles
   - reset dug soft soil
   - keep the host rotation continuing; do not reset host to Springtail unless the rotation naturally lands there

## Room Progression

- Reaching `E` loads the next room.
- After Room 3, show a win screen that says: `You restored the soil.`
- Win screen includes a restart option.
- Restart resets to Room 1 and Springtail.

## Controls

- Left/right: move.
- Space:
  - Beetle jump.
  - Springtail dash if moving left or right.
  - Earthworm does nothing.
- Down + direction:
  - Earthworm tunnels through soft soil.
- R: restart current room.
- Escape: quit.
- Enter on win screen: restart full game.

## Code Style Requirements

- Use clearly commented modules.
- Use plain names.
- Keep code readable for a non-developer.
- Prefer small classes and functions.
- Avoid clever abstractions.
- Add comments explaining tile parsing, collision checks, host abilities, death rotation, corpse terrain mutation, room reset, and asset fallback behavior.

## Main Workflow

1. Player launches the game.
2. Room 1 loads.
3. Player starts as Springtail at `P`.
4. Player moves using host-specific abilities.
5. Player reaches exit or dies on hazards.
6. If player dies:
   - corpse terrain mutates the room
   - deaths-in-room increases
   - host rotates
   - player respawns at `P`
7. If deaths-in-room reaches 3:
   - room resets to original layout
   - host rotation continues
8. If player reaches exit:
   - next room loads
9. After Room 3:
   - show win screen
10. Player can restart.

- Confirm Springtail passes through `G`.

- Confirm Earthworm digs `S`.

- Confirm death rotation.
- Confirm corpse terrain persists.
- Confirm room reset after three deaths.
- Confirm exits and win screen.
